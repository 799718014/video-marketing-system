## 二、功能缺失与建议

### 🔴 严重缺失（建议优先补充）

| 缺失项 | 影响 | 建议方案 |
|--------|------|---------|
| **音频拼接** | 当前只处理视频轨道，合并后视频无声音 | 在 merge_videos 中添加音频合并逻辑，使用 moviepy 的 audio 处理能力 |
| **断点续传/任务恢复** | 任务中断后无法从断点继续 | 将任务状态持久化到数据库（当前仅内存存储） |
| **进度百分比** | 前端无法显示精确进度 | 添加 progress 字段，计算已完成时长/总时长 |

### ⚠️ 潜在问题

| 问题 | 影响 | 建议方案 |
|------|------|---------|
| **帧率不一致** | 不同片段帧率不同可能导致合并失败 | 在合并前统一帧率，使用 moviepy 的 fps 参数 |
| **分辨率对齐** | 不同分辨率片段拼接效果差 | 在合并前统一分辨率，添加缩放处理 |
| **内存泄漏风险** | VideoFileClip 未正确清理可能导致内存问题 | 确保 close() 在 finally 块中调用 |
| **临时文件清理** | 程序异常退出时临时文件残留 | 使用更健壮的清理机制（atexit注册） |

### 💡 功能增强建议

| 建议项 | 价值 | 实现难度 |
|--------|------|---------|
| **进度回调/WebSocket** | 实时推送进度到前端 | 中等 |
| **更多转场效果** | 丰富视觉体验（溶解、滑动等） | 低 |
| **片段预览图** | 帮助用户了解各片段内容 | 低 |
| **任务优先级队列** | 高优先级任务优先处理 | 中等 |
| **生成参数调优** | 不同片段使用不同参数 | 低 |

## 四、代码质量问题

### 1. 重复的路由定义

**位置**: `batch_video.py:218` 和 `batch_video.py:325`

**问题**: `retry_segments` 函数被定义了两次，第二个是"修正版"

**影响**: 后定义会覆盖前面的，但容易造成混淆

**建议**: 删除第一个定义，保留修正版

### 2. 内存存储不持久

**位置**: `batch_video_service.py:14`

```python
_batch_tasks: Dict[str, BatchVideoTask] = {}
```

**问题**: 任务仅存储在内存中，服务重启后丢失

**建议**: 使用数据库或文件持久化

### 3. 缺少日志级别控制

**位置**: 多处 `logger.info/error/warning`

**问题**: 无法根据环境动态调整日志级别

**建议**: 使用配置文件控制日志级别

## 七、结论

1. **功能完整性**: 缺少音频拼接和任务持久化
